Operating Systems Terminology

TERM DEFINITION
Operating system The software that controls the overall operation of a computer. It provides the means by which a user can store and retrieve files, provides the interface by which a user can request the execution of programs, and provides the environment necessary to execute the programs requested.
Job queue A storage organization in which jobs are ordered based on a variety of prioritizations.
FIFO Abbreviation of first-in, first-out.
Interactive processing An operating system that allows a program being executed to carry on a dialogue with the user through remote terminals.
Time-sharing Operating systems that provide service to multiple users at the same time.
Multiprogramming A technique of implementing time-sharing by dividing time into intervals and then restricting the execution of each job to only one interval at a time.
Application software A broad category of a machine's software that consists of the programs for performing asks particular to the machine's utilization.
System software A broad category of a machine's software that performs the tasks that are common to computer systems in general by providing the infrastructure that the application software requires.
Utility software A category of system software that consists of software units that extend (or perhaps customize) the capabilities of the operating system.
Graphical user interface (GUI) A graphics-based operating system that manipulates objects, such as files and programs, by representing them pictorially on the display as icons.
Shell An older form of user interface that communicates with users through textual messages using a keyboard and monitor screen.
Kernel The internal part of an operating system contains the software components that perform the very basic functions required by the computer installation.
Device manager The software units that communicate with the controllers (or at times, directly with peripheral devices) to carry out operations on the peripheral devices attached to the machine.
File manager A unit that coordinates the use of the machine's mass storage facilities by maintaining records of all the files stored in mass storage, including where each file is located, which users are allowed to access the various files, and which portions of mass storage are available for new files or extensions to existing files.
Memory manager A component of a kernel which is charged with the task of coordinating the machine's use of main memory.
Virtual memory A "fictional" memory space created by paging.
Boot strapping A procedure for starting operating systems that is performed by a computer each time it is turned on.
ROM A small portion of a computer's main memory where the CPU expects to find its initial program that is constructed from special nonvolatile memory cells. The contents can be read but not altered.
Deadlock The condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
Spooling A technique of holding data for output at a later but more convenient time.
Super user (or administrator) A person who gains highly privileged access to the operating system by identifying him- or herself as the administrator (usually by name and password) during the login procedure.
Multi-factor authentication A type of authentication that requires multiple forms of evidence in order to gain access to computer accounts.
